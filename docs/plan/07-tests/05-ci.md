# 07-tests / 05 — CI-friendly validation commands

Status: `prog`

## Goal

Make validation clear for humans, Codex, and future CI.

## Required commands

- Fast setup.
- Fast tests.
- Optional slow enumeration tests.
- Full local check.

## Done means

- `make test` and `scripts/test.*` run the fast suite.
- Slow tests can be enabled explicitly.
- Command output shows the Python executable in use.

## Progress notes

- The Makefile provides CI-friendly local commands: `make setup`, `make test`,
  `make supports`/`bundles`/`labelings`/`reduced`, and aggregate `make check`,
  all using `.venv/bin/python`. (The legacy `legacy-count` target was removed
  in `docs/plan/02-cleanup/02-legacy.md`.)
- `.codex/setup.sh` and `.codex/maintenance.sh` exist for Codex Cloud setup and
  cached-environment maintenance.
- Bash/Linux/WSL validation scripts provide the non-Make path:
  `./scripts/setup.sh`, `./scripts/test.sh`, `./scripts/lint.sh`,
  `./scripts/check.sh`, and `./scripts/clean.sh`.
- `make setup`, `make test`, `make lint`, `make check`, and `make clean`
  delegate to the Bash scripts.
- Native PowerShell validation scripts provide the Windows path:
  `.\scripts\setup.ps1`, `.\scripts\test.ps1`, `.\scripts\lint.ps1`,
  `.\scripts\check.ps1`, and `.\scripts\clean.ps1`.
- The Windows path was verified natively through clean setup, test, lint, full
  check, cleanup, wrong-directory guards, and repeat setup/cleanup runs.
- Explicit slow-test selection is still missing, so this remains `prog`.
- 2026-07-11: Added repository-carried change-aware validation with
  `validation/impact.toml`, `scripts/validate_changes.py`, `make
  validate-changed`, selector tests, and a stable GitHub Actions validation job
  that uses the same selector. Documentation-only changes now run a lightweight
  docs profile; source/test changes run lint plus pytest; tooling, CI,
  dependency, validation-policy, and unknown changes escalate to `make check`.
- 2026-07-11: Hardened the change-aware selector after PR review. Validation
  machinery paths are now pinned to full validation even if a policy edit tries
  to downgrade them, the GitHub Actions workflow performs the same full-profile
  preflight before invoking a PR-edited selector, and the plan-index checker now
  rejects missing group headings and cross-group task links.

- 2026-07-12: Kept linting on the repository-owned `scripts/lint.*` path
  instead of adding a task-local Ruff dependency; `make check` delegates back
  to `scripts/check.sh` so wrapper validation remains covered by the primary
  validation path.
